# Agent 工作约定

## 变更记录与版本号

- 每次修改代码、配置、部署脚本或用户文档时，都必须同步更新 `CHANGELOG.md`。
- `CHANGELOG.md` 遵循 Keep a Changelog 风格，条目应放入 `新增`、`改进`、`修复`、`安全`、`运维` 等合适分类。
- 版本号遵循语义化版本：
  - 修复 bug、性能优化、兼容性优化：提升 patch 版本，例如 `1.6.5` -> `1.6.6`。
  - 新增向后兼容能力：提升 minor 版本，例如 `1.6.x` -> `1.7.0`。
  - 破坏性变更：提升 major 版本，例如 `1.x.x` -> `2.0.0`。
- 不建议每次纯内部整理、注释、格式化或测试辅助改动都提升正式版本号；这些改动仍需写入 `CHANGELOG.md`，但可合并到下一次发布级版本。
- 提升版本号时，至少同步更新：
  - `CHANGELOG.md` 顶部最新版本条目。
  - `README.md` 中的当前版本说明。
- 本项目的 Docker 发布流程会从 `CHANGELOG.md` 顶部最新版本提取镜像标签，因此发布前必须确认顶部版本号准确。
