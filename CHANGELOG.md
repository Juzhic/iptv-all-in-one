# 更新日志

所有重要更改都会记录在此文件中。

格式基于 [Keep a Changelog](https://keepachangelog.com/zh-CN/1.0.0/)，
版本号遵循 [语义化版本](https://semver.org/lang/zh-CN/)。

**重要说明：每次代码修改都必须将更新内容写入本文件；只有发布级或用户可见变更才提升版本号。**

## [1.7.7] - 2026-06-29

### 修复
- 修复定期检测成功检测单条记录后仍可能因质量评估入口缺失而中断整轮检测的问题，避免检测概览出现“检测数 0、通过 1、明细 0”的错位统计，并确保失败次数与阈值软删除流程继续执行。
- 定期检测单条结果后处理异常时改为按失败明细记录并继续本轮检测，异常或超时收尾时的检测数会按已处理结果兜底计算。

### 改进
- 扫描结果来源分组新增“最近检测”和“最近入库”两个时间口径，并保留原 `last_updated` 字段兼容旧调用，避免把单个频道检测时间误读为整组更新时间。

## [1.7.6] - 2026-06-29

### 修复
- 修复部署包 `deploy/database/__init__.py` 未同步导出 `_evaluate_quality`、软删除复活和 IP 扫描相关数据库函数，导致使用 deploy 包运行时频道扫描或检测流程仍可能报 `module 'database' has no attribute ...` 的问题。
- 扫描收益统计的质量分档增加数据库入口兜底导入，避免数据库包入口漏导出时直接中断整轮扫描。

### 改进
- 优化频道扫描页失败态展示：失败原因改为独立告警块展示，扫描概览只保留时间和阶段统计，避免错误徽标和长错误文案在深色界面中显得突兀。
- 日志面板与扫描统计卡片改为使用 TDesign 主题变量，减少深色主题下亮色渐变卡片和大面积紫色日志背景造成的视觉割裂。

## [1.7.5] - 2026-06-29

### 修复
- 修复 Docker 部署中 `database` 包入口未导出质量评估、频道复活和 IP 扫描数据库函数，导致频道扫描、检测复活或 IP 扫描相关接口返回 `module 'database' has no attribute ...` 的问题。
- 前端新增运行时 SSE 能力自动判断：多线程 WSGI 环境自动使用 SSE，旧 Compose 或面板仍覆盖为单同步 worker 时自动降级短轮询，避免长连接占满唯一 worker。
- 新增 `/api/ip-scan/logs` 增量日志接口，让 IP 扫描在短轮询模式下仍能刷新日志。

### 运维
- 同步 `deploy/Dockerfile` 的 Gunicorn 默认启动参数为 `gthread` 多线程模式，避免使用部署产物构建镜像时仍保留旧的单同步 worker 配置。
- Docker Compose 部署文件显式写入应用容器的 `gthread` 启动命令，避免面板重部署时把 CMD 恢复为旧的单同步 worker 参数。

## [1.7.4] - 2026-06-29

## [1.7.0] - 2026-06-28

### 新增
- 新增测绘平台/画像收益统计表 `scan_yield_stats`，记录每轮扫描中平台、画像、省份维度的目标量、API 命中、实际探测、频道提取、清洗、去重、候选、快筛通过、深度检测通过和质量分档数量。
- 新增 `/api/scan/yield-stats` 接口和前端 API 封装，用于查询已持久化的平台/画像扫描收益数据。
- 扫描配置新增 `deep_check_duration`、`deep_check_min_bytes`、`deep_check_request_timeout`，用于统一控制扫描板块深度检测采样时长、最小采样字节数和请求超时。
- 扫描配置页新增深测时长、深测采样和深测超时控件，便于直接调整扫描板块统一深度检测口径。

### 改进
- 测绘平台基础查询与质量优先画像查询会为候选频道打上内部收益标识，扫描主流程在快筛和深度检测结束后回填真实产出，便于后续按平台和画像动态调整额度。
- 扫描后的后台深度检测、持久化结果手动重检和定期检测统一通过同一个 `run_deep_check()` 入口执行，避免扫描板块内部检测口径分散。
- 删除或清理扫描历史时同步清理对应的平台/画像收益统计记录，避免历史数据残留。
